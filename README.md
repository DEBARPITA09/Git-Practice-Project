# Git Collaboration Practice Project

## 📌 Overview

This project was created to understand and practice collaborative development using Git and GitHub.

The main objective was not just building a web project, but learning how multiple developers can work together efficiently using version control.

---

## 🎯 Purpose

- To understand how Git tracks changes
- To simulate real-world team collaboration
- To experience how GitHub handles remote repositories

---

## 👩‍💻 Collaboration Setup

This project was developed by two collaborators.

Both contributors worked on different features and practiced:

- Creating branches
- Committing changes
- Pushing to remote
- Pulling updates
- Reviewing changes

---

## 🔄 Git Concepts Practiced

- `git clone`
- `git add`
- `git commit`
- `git push`
- `git pull`
- Remote repository synchronization

---

## 🛠️ Tech Used

- HTML
- CSS
- Git
- GitHub

---

## 📚 Key Learnings

- Importance of pulling latest changes before starting work
- How version control prevents overwriting others' work
- How GitHub manages collaboration through shared repositories
- How proper commit messages improve project clarity
  
---

## 🚀 Outcome

This project helped in understanding real-world development workflows and strengthened practical knowledge of Git and GitHub collaboration.
